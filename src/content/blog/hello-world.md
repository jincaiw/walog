## 为什么写博客

从开始学编程到现在，已经好几年了。这些年里，我读过无数篇技术博客，也从各种文档和社区讨论中受益良多。但自己一直没有一个系统记录和分享的地方。

写博客这件事，对我而言有以下几个意义：

### 1. 加深理解

费曼学习法告诉我们：最好的学习方式是教给别人。写文章本质上就是在教——你需要把一个知识点用清晰的语言表述出来。这个过程会逼你想清楚很多细节，那些「以为自己懂了」的知识盲区会自然浮现。

### 2. 构建知识体系

零散的笔记和书签很难形成体系。写博客迫使你把相关的知识点串联起来，形成结构化的知识网络。久而久之，这就是你自己的知识库。

### 3. 记录成长

回头看看自己一年前写的代码和文章，往往能清楚地看到进步。这种记录本身就是一种正反馈。

## 博客的技术选择

这个博客基于以下技术构建：

- **Astro** — 静态站点生成器，天然支持 Markdown，JS 交付量极小
- **Tailwind CSS 4** — 实用优先的 CSS 框架
- **GitHub Pages** — 零成本部署
- **自定义域名** —  mujizi.com

所有文章使用 Markdown 编写，存储在 Git 仓库中，不依赖数据库。

## 写什么

这个博客主要会写以下内容：

- 编程实践与踩坑记录
- 系统设计与架构思考
- 工具和效率方法
- 偶尔的非技术随笔

## 结语

希望这个博客能持续写下去。如果某篇文章恰好对你有帮助，那就是额外的收获了。
