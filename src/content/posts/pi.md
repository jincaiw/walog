# Pi Agent 是什么？

> Pi —— 一个极简而极致可扩展的终端编码助手（terminal coding harness），它的哲学是：**适配你的工作流，而不是让你去适配它**。

---

## 一句话总结

**Pi 是一个以 AI 为核心的终端编码中枢**。它不强制你使用某种固定的工作方式，而是通过一套可扩展的机制（Extensions、Skills、Themes、Packages），让你按照自己的习惯来搭建开发流程。

---

## 核心能力

### 1. 工具调用引擎

Pi 最本质的能力是给 AI 模型提供四件套核心工具，让模型直接操作你的文件系统和代码：

| 工具 | 用途 |
|------|------|
| `read` | 读取文件内容 |
| `write` | 创建/覆写文件 |
| `edit` | 精确修改文件（基于文本匹配） |
| `bash` | 执行终端命令 |

在此基础上，通过 Skills、Extensions 和 MCP 还能扩展出更多能力（搜索、LSP 导航、AST 语义匹配等）。

### 2. 多种运行模式

Pi 适应不同的使用场景，不局限于一种交互方式：

| 模式 | 适用场景 |
|------|----------|
| **交互模式**（默认） | 日常开发，人类在环中协作 |
| **打印模式**（`-p`） | 脚本调用、管道集成 |
| **JSON 模式**（`--mode json`） | 前端 UI 集成，流式事件 |
| **RPC 模式**（`--mode rpc`） | 非 Node.js 进程集成，JSONL 协议 |
| **SDK 模式** | 嵌入到你自己的应用中 |

### 3. 极致可扩展性

这是 Pi 区别于其他编码助手最显著的特点。它没有内置大量功能，而是提供了灵活的扩展机制：

| 扩展方式 | 说明 |
|----------|------|
| **Extensions** | 用 TypeScript 编写功能扩展，可以添加自定义工具、事件监听等 |
| **Skills** | 把任意 CLI 工具包装成 AI 可调用的能力，本质是「CLI 工具 + README 规范」 |
| **Prompt Templates** | 自定义提示词模板 |
| **Themes** | 自定义终端主题 |
| **Pi Packages** | 通过 npm 或 git 分享和安装上述所有资源 |

> 💡 **设计哲学**：其他工具内置的功能（sub-agents、plan mode、权限弹窗、任务列表），Pi 鼓励你用扩展自己构建，或者安装第三方包。这让核心保持最小化，同时保证你不会被工具的工作方式束缚。

### 4. 灵活的集成能力

- 支持多种 AI 提供商（Anthropic、OpenAI、Google 等）
- 支持 API Key 和订阅登录
- 通过 Session 系统管理对话历史，支持分支和压缩
- 支持 AGENTS.md / CLAUDE.md 上下文文件
- 支持 tmux 集成，实现完整的终端可观测性

## 它不适合什么？

| 场景 | 为什么不合适 |
|------|-------------|
| **通用 AI 聊天** | Pi 的设计目标就是编码场景，不是通用聊天工具 |
| **不想动手配置** | 极致可扩展意味着开箱时功能相对精简，需要一些配置 |
| **需要完整 IDE 体验** | Pi 是终端工具，没有图形界面和鼠标操作 |

---

## 它和别的工具比有什么不同？

| 特性 | Pi | Cursor / Windsurf | GitHub Copilot |
|------|----|-------------------|----------------|
| 界面 | 终端 TUI | 桌面 IDE | VS Code 插件 |
| 扩展性 | 极致（Extensions + Skills + Packages） | 有限 | 有限 |
| 定制程度 | 完全自定义 | 有限配置 | 有限配置 |
| 内置功能 | 极少（保持核心最小） | 丰富（开箱即用） | 丰富（开箱即用） |
| 使用成本 | 自带 API Key / 订阅 | 订阅制 | 订阅制 |
| 哲学 | 适配你的工作流 | 你适配工具 | 你适配工具 |

---

## 典型使用场景

1. **代码重构与修改** — 利用 `read`/`edit`/`write` 精确操作代码
2. **项目分析与理解** — 读取代码库、生成文档、分析架构
3. **自动化脚本** — 通过 `-p` 模式嵌入到 shell 管道中
4. **定制工作流** — 用 Skills 把内部 CLI 工具封装成 AI 能力
5. **构建自己的 AI 编码产品** — 通过 SDK 或 RPC 模式嵌入到你的应用中

---

## 总结

**Pi 不是一个"开箱即用"的 AI 编码助手，而是一个让你**自己搭建 **AI 编码工作流的平台。** 它适合那些：

- 清楚自己想要什么工作流
- 愿意花时间定制环境
- 不想被工具厂商的决策绑架

的人。

如果你是这样的开发者，Pi 可能是最适合你的工具。🚀
